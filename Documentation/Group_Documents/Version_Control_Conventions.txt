Computer Science Final Project - Versionsstyring

Versionstyring gøres på GitHub hvor vi har et repository til vores løsning. Heri har vi en master branch, som hovedsageligt eksisterer 
til at lave release på. Dernæst development branch (som er vores main branch) som vi skaber feature/bug fixing branches fra, og merger feature/bugfix
branches ind i (når de er blevet kvalitetssikret). Til sidst har vi vores feature/bug fixing branches, som er der vi udvikler og fixer.

Alle branches skal have en issue oprettet i issue trackeren, og alle feature branches skal også have en use case eller user story de tager udgangspunkt i.
Konflikter i med developer branchen skal ordnes i feature/bugfix branchen frem for på developer.
De eneste ting som må direkte committes til developer branch er dokumentation.
 
Versionsstyring Guidelines
 - Når der udvikles skal det være i branches som er lavet fra development
 - Alle branches skal være tied til et Issue, og branchens navn skal være det samme som Issuet
 - Når man merger skal det være med development, er der blevet lavet noget på development skal development merges ind i branchen før man kan merge den ind i development
